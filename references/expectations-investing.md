# 预期差投资（Expectations Investing）——报告的分析主线

核心思想（Rappaport & Mauboussin）：股价不是对未来的预测，而是**一组已定价的预期**。研究的产出不是"公司好不好"，而是"**市场隐含的预期 vs 我的预期，差在哪、凭什么、何时修正**"。没有可陈述的预期差（variant view），报告就只是资料汇编。

## 1. 三步框架

**第一步：解读市场预期（不预测，只解码）**
- 用反向 DCF（`dcf.py` 的 `reverse` 块）解出现价隐含的稳态 FCF、所需营收和隐含 CAGR。
- 用 PVGO 分解（`dcf.py` 的 `pvgo` 块）：`现价 = 零增长价值(E/r) + 增长期权价值(PVGO)`，算出**现价中有多少百分比是在为"尚未发生的增长"付费**。
- 把隐含预期翻译成可感知参照系："隐含未来 10 年 X% 增速——历史上市值超过 $Y 后做到这一点的公司有 N 家"（对照 `base-rates.md`）。

**第二步：建立自己的预期（用外部视角纪律化）**
- 每个关键驱动（收入、利润率、再投资回报）先查 `base-rates.md` 的历史分布，把自己的假设放进分布里定位（"我假设的增速处在历史第 80 分位"）。
- 超越 base rate 的假设必须给出**结构性理由**（护城河来源、行业渗透曲线位置、可验证的领先指标），"管理层有信心"不算理由。

**第三步：陈述预期差与修正路径**
- 分歧写成可证伪命题："市场认为 X，我认为 Y，因为证据 Z；验证信号是 W（时点）"。
- 识别**预期修正的触发器**：哪个季度的哪个 KPI 会迫使市场改变定价；没有触发器的低估可以低估很久。

## 3. 独立观点检验（硬性）

成稿前回答，答不出则不得给出"买入/减持"（只能"观望"）：

1. 我和共识的**最大单一分歧**是什么？一句话。
2. 这个分歧**为什么存在**（市场犯了什么可识别的错：外推、锚定、时间视界套利、结构变化未察觉）？
3. 如果我错了，**最早在什么时候、通过什么数据**能发现？

## 4. 与其他模块的关系

- 反向 DCF 从"估值方法之一"升格为**第一章的开篇框架**；正向 DCF/EPV/相对估值负责回答"我的预期值多少钱"。
- 预期差方向必须与第 8 节标定标签自洽：标签"低估"但 Gap 表显示我的预期低于市场隐含 → 矛盾，必须回头查哪边错了。
- 财报模式：每次财报把"实际 vs 市场预期 vs 我的预期"三线对账，预期差被验证/证伪要显式记账（论点计分）。
