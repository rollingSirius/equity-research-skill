# 估值方法

目标：用**至少三种**互相独立的方法给出合理价值区间，汇总成表并算出相对现价的隐含空间。多方法的意义在于交叉验证——区间相近则结论更可信；分歧大要解释为什么（通常是增长/盈利假设或折现率差异），并说明采信哪个。

原则：
- 每种方法**显式列出关键假设**，假设比小数点重要。
- **所有 DCF 类计算（正向/反向/敏感性/加权）一律用 `scripts/dcf.py` 或等效脚本执行，禁止心算**。敏感性表一张 3×3 就是 9 次完整折现，心算必错。假设以 JSON 落盘，本身就是留档。
- 估值是区间不是点；结论标签由第 8 节的预注册规则映射产出，不允许临场发挥。

## 1. 相对估值（可比公司 + 历史区间）

常用倍数，按公司类型选择：
- **P/E（TTM 与 Forward）**：盈利稳定的公司。
- **EV/EBITDA**：资本结构差异大、折旧重的公司，跨公司更可比。
- **P/S**：高增长但未稳定盈利的公司。
- **PEG = P/E ÷ 盈利增速**：成长股，>1 偏贵、<1 偏便宜（粗略）。
- 高增长软件/订阅：**EV/Sales、EV/ARR**；亦看 Rule of 40。

做法：
1. 取公司自身**历史倍数区间**（如近 5 年 P/E 的 25–75 分位与中位），看当前位置。
2. 选 3–6 家**可比公司**对比同一倍数（A股/港股对照系选取见 `markets-cn-hk.md` 第 6 节）。
3. "合理倍数 × 对应财务指标" = 每股合理价值区间。

相对估值表：

| 倍数 | 当前 | 自身历史中枢 | 可比公司中位 | 选用合理倍数 | 隐含每股价值 |
|---|---|---|---|---|---|

## 2. 绝对估值：DCF（正向）

三阶段自由现金流折现：显性期 5–10 年 + 增速衰减期 3–6 年 + 永续。衰减期让末年增速线性降至永续 g，避免"高增长直接撞上 3% 永续"的断崖（那会系统性低估仍在成长的公司，或诱使你抬高永续 g 作弊）。

关键假设（**必须列出**）：
- 收入增速路径（逐年，反映周期/渗透/见顶）。
- FCF 利润率路径（向稳态收敛；注意烧钱期为负值也要如实建模）。
- **WACC**：CAPM 估权益成本，按资本结构加权，写明取值。
- **永续增长 g**：一般 2–3%（不超过长期名义 GDP），高于此需强论证。
- 股本稀释：SBC 重或有股票并购习惯的公司，显性期按年稀释率建模。

执行：把假设写进 JSON，跑 `python scripts/dcf.py --config <file>`。输出企业价值 → 减净债 → ÷ 期末股本 = 每股内在价值，并附带退出 P/FCF 倍数用于自检（若隐含退出倍数明显高于成熟同业，说明永续假设在偷偷乐观）。

**敏感性分析（必做）**：WACC × 永续 g 的 3×3 表（脚本内置）；成长股再做一张 收入增速 × 稳态利润率。

## 3. 三要素估值法：资产重置 · 盈利能力价值 EPV · franchise 成长（Graham–Greenwald）

**为什么需要它**：DCF 的估值几乎全由最难预测的「未来增长率」决定（增速差 2%，估值差 50%+），对多数公司近乎玄学。三要素法反过来「从最确定到最不确定」自下而上叠加：资产负债表锁定底价 → 当期利润算「不增长也值多少」→ 最后才谨慎为增长付费；并把「护城河」从定性变成可计算的数字。**与本手册其他方法冲突时，以 Greenwald《价值投资：从格雷厄姆到巴菲特》为准。**

**何时用**：业务简单、利润稳定的消费/护城河型公司（最锋利）；周期/叙事型高估值股锚定估值下限、给 DCF 证伪；A股/港股缺乏可靠前瞻一致预期时的主力法；与 DCF 交叉验证。**不适用**：银行/保险（用 P/B–ROE、P/EV）、尚未盈利的早期成长股（EPV 无意义，回反向 DCF / 情景加权）。

**第一层 · 资产重置价值（底价）**：对手从零复制资产要花多少（重置成本）/清算能收回多少。资产负债表逐项「挤水分」后减总负债：

| 资产类别 | 折扣率 | 资产类别 | 折扣率 |
|---|---|---|---|
| 现金及等价物 | 100% | 机器设备 | 20–30% |
| 短期可变现证券 | 95–100% | 厂房 | 40–60% |
| 应收账款 | 80–90% | 商业地产/土地 | 按市价评估 |
| 存货（原料 / 成品） | 80–90% / 50–70% | 商誉 | 0–20%（多按 0） |

隐性资产（研发/团队/品牌）第一层不计，留待 EPV 体现；轻资产公司此层可能接近 0，价值几乎全在 EPV/成长。用途=绝对底线：破净且行业未衰退可能是烟蒂，但破净常有原因（薄利/夕阳/毁灭价值）。

**第二层 · 盈利能力价值 EPV（核心）**：假设永续零增长，`EPV = 常态化盈利 ÷ 资本成本`。
- 常态化盈利：剔除非经常损益，加回超出维持性资本开支的 growth capex。**口径默认 NOPAT = EBIT×(1−税率)**（剔除资本结构影响；NOPAT÷WACC 得企业价值，再减净债、加过剩现金得权益）；**数据不足或计算复杂时退回净利润**（此时资本成本视作权益成本，直接得权益价值，不再减净债）。利润波动大的公司取 3–5 年均值。
- **资本成本与第 2 节 DCF 的 WACC 口径保持一致**（CAPM 估权益成本、按资本结构加权，写明取值）；无法建企业层 WACC 时才退回「无风险利率+股权风险溢价」简化档（美≈8%、A≈6%）。
- EPV 已含资产赚钱能力，**不与第一层相加**；净资产是对比锚，不是叠加项。

**护城河的财务验证**（本法独有的彩蛋）：EPV 与调整后净资产并比，且**看多年趋势**——
- EPV ≈ 净资产 → 无/弱护城河的辛苦生意；EPV < 净资产 → 毁灭价值（ROIC<资本成本），规避；**EPV ≫ 净资产且长期稳定 → 护城河的财务度量**，差额=特许经营价值。单年暴涨多是景气非壁垒。须与第三章「护城河评分表」交叉印证，定量与定性不一致时显式解释。

**第三层 · 成长价值（仅在护城河内计）**：增长只有在 **ROIIC（增量投入资本回报率）> 资本成本**时才创造价值；否则增长越快、价值毁灭越多，成长价值记 0 或负。
- **默认严格式（Greenwald franchise-growth）**：`价值 = 常态化盈利 ×(1 − g/ROIIC) ÷ (资本成本 − g)`（要求 g<资本成本才收敛；ROIIC=资本成本 → 退化为 EPV；ROIIC<资本成本 → 低于 EPV）。
- **数据不足（缺 ROIIC 或 g≥资本成本无法收敛）时退回简化式**：`成长调整价值 ≈ EPV ×(1 + 保守增长率)`（只取一年缓冲，不做 5/10 年激进外推）。
- 保守增长率：历史 5 年净利润 CAGR 与宏观锚定（美≈6%、A≈9%，再加行业增速）**取其低**。
- 哲学：绝不为不确定增长付高溢价，让成长成为安全边际外的免费礼物。

**执行（禁止心算）**：假设写进 JSON 的 `epv` 块，跑 `python scripts/dcf.py --config <file>`——脚本输出 EPV 权益价值/每股、EPV÷净资产护城河判定与多年趋势、franchise 成长价值（自动在无法收敛/缺 ROIIC 时兜底简化式）、买点阶梯（底价<EPV<成长调整）与现价所处区间。`epv` 块字段：`earnings_basis`(NOPAT/net_income)、`normalized_earnings`、`coc`(=WACC)、`net_debt`、`excess_cash`、`shares`、`asset_value`、`price`、`growth:{g, roiic, mode}`、可选 `asset_series`(多年趋势)。

**结论映射**：三层给出阶梯，按现价分层——≤底价：极端安全（烟蒂，须排除价值毁灭陷阱）；底价~EPV：**黄金击球区**（买到当前盈利，成长白送）；EPV~成长调整：合理，安全边际薄；>成长调整：偏贵。可令 range_low=EPV、range_high=成长调整并入第 7 节的综合区间（确定性高的消费/护城河股尤其适用），**最终标签仍由第 8 节标定规则产出**。

## 4. 反向 DCF（Reverse DCF）——优先作为框架性方法

正向 DCF 问"公司值多少钱"，反向 DCF 问"**现价假设了什么**"——后者更抗假设操纵，因为它不需要你预测未来，只需要你判断市场隐含的未来是否可信。对高估值成长股与叙事股，反向 DCF 应作为估值章节的**开篇框架**。

做法（脚本 `reverse` 模块）：
1. 由现价算出 EV。
2. 设定建设期（3–5 年）FCF 路径（可参考当前实际烧钱/产出水平外推）。
3. 解出：稳态首年需要多大 FCF 才能支撑 EV。
4. 把该 FCF 换算成"稳态利润率 × 所需营收"，再算隐含营收 CAGR。

呈现要点：把隐含要求翻译成**可感知的参照系**（"需要 X 年内达到某巨头级别的 FCF""隐含连续 N 年 Y% 增速，历史上做到的公司有几家"）。结论句式："市场不是在给××定价，而是在给××定价"。

## 5. 情景加权公允价值（Probability-Weighted Fair Value）——作为综合方法

当业务结果分布很宽（新技术、平台转型、强政策敞口），单一 DCF 的中间假设没有意义。做法：

1. 定义 3 个情景（熊/基准/牛），每个情景是一套**内部自洽**的假设（收入路径 + 利润率路径 + 稀释率），不是简单把基准 ±20%。熊市情景要问"理性管理层届时会怎么做"（如砍掉亏损业务），不要把当前的烧钱线性外推到永续——那不是熊市，是漫画。
2. 赋主观概率（和为 1），**写出定概率的依据**。
3. 每个情景独立跑 DCF，概率加权得公允价值；脚本一次完成。
4. **稳健性检验（必做）**：把概率推到对你结论最不利的极端组合，看加权值是否仍支持同一标签。若结论翻转，如实写"结论对概率假设敏感"，降低置信度。
5. 与 Morningstar 等第三方的情景框架对照（若可得）：分歧点通常在概率与终局体量，逐项对账。

## 6. 其他适用方法（按行业选其一以上）

- **SOTP 分部估值**：多业务公司分部估值加总，减净债与总部成本。注意：SOTP 用市场倍数，内嵌当下市场情绪，与 DCF 的分歧本身是信息（市场溢价的定量化）。
- **Morningstar 公允价值**：独立第三方锚点，标日期、星级、不确定性评级；区分分析师公允价值与量化评级（后者含义弱，见 `markets-cn-hk.md` 第 3 节）。
- **行业特定指标**：半导体/周期用归一化盈利 × 中枢倍数；SaaS 用 EV/ARR、NRR；银行/保险用 P/B–ROE、P/EV；资源/地产用 NAV。

## 8. 结论标定规则（预注册，不得临场发挥）

**为什么需要这一节**：同一组数字，两次运行不应给出不同标签。标签由规则产出；分析师若要偏离，必须显式写出偏离理由，禁止静默改标签。

**估值标签**（设综合区间为 [L, H]，现价 P；±15% 缓冲带承认区间本身的不精确）：

| 条件 | 标签 |
|---|---|
| P < 0.50 × L | 显著低估 |
| 0.50L ≤ P < 0.85L | 低估 |
| 0.85L ≤ P ≤ 1.15H | 合理 |
| 1.15H < P ≤ 1.50H | 高估 |
| P > 1.50 × H | 显著高估 |

**动作矩阵**（标签 × 不确定性等级；不确定性优先取 Morningstar 评级，缺失则自评低/中/高/极高并写明依据：业务可预测性、经营与财务杠杆、事件风险）：

| 标签 \ 不确定性 | 低 / 中 | 高 / 极高 |
|---|---|---|
| 显著低估 | 买入 | 分批买入（且要求 P < 0.70L 方可首建） |
| 低估 | 买入/分批买入 | 小仓位试探或继续等待 |
| 合理 | 观望（持有者持有） | 观望 |
| 高估 | 减持/规避 | 规避 |
| 显著高估 | 规避，持有者减持 | 规避，持有者减持；**默认不建议做空**（借券成本、逼空与尾部风险另议） |

**否决项（一票降级）**：治理红旗 ≥2 项严重级（关联交易非公允、信披劣迹、频繁损害小股东的资本运作等）或财报可信度存疑 → 无论估值标签，动作最高只到"观望"；护城河判"无"且标签为高估 → 直接"规避"。双重上市标的按 `markets-cn-hk.md` 第 5 节**分市场分别标定**。

**一致性要求**：第一章速览与第九章结论的标签、动作必须与本规则可复算地一致；脚本 `dcf.py` 会在给定 `range_low/range_high` 时自动输出标签建议。
