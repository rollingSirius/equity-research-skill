# 输出格式与可读性规范

目标：深度不等于难读。机构研报的可读性来自**结论前置、信息分层、数字规范、视觉锚点**。撰写正文前必读本文件。

## 1. 报告头部（三件套）

依次为：结论框 → Tearsheet → 预期差 Gap 表。读者 60 秒内应获得完整决策信息。

**结论框**（引用块，放标题之后第一屏）：

> **结论：低估 → 分批买入**（标签由标定规则产出｜财报可信度 A｜置信度 中高）
> 一句话论点：市场把 X 定价成 Y，但证据 Z 显示……
> 综合估值区间 $L–$H（现价 $P，隐含空间 +a% ~ +b%）｜最大风险：……

**Tearsheet**（一页速览表）：现价/市值/52周区间/关键倍数/护城河/财报可信度/催化剂 Top1/证伪信号 Top1，每行带来源+时间戳。

## 2. 章节写法

- 每章第一行：**「本章要点：」+ ≤2 句结论**。先给答案再给论证，读者可只读要点扫完全文。
- 段落 ≤5 行；超过就拆或改表格。连续 3 段没有数字的分析要警惕——多半是空话。
- "事实"与"我的判断"用标记显式分离；判断句给出依据编号或数据。
- 图表就近原则：表格紧跟引用它的论述，不集中堆在章末。

## 3. 数字规范

- 千分位分隔；货币符号 + 单位一次性声明（"以下未注明均为 $B"）；百分比保留 1 位小数，估值每股保留整数或 1 位。
- **精度匹配不确定性**：DCF 输出写 "$185–$240" 而不是 "$212.47"——假装精确是伪严谨。
- 同比/环比用 +/− 号；负数不用括号（中文语境）。
- 每张表格列出"来源/时间"列或表注。

## 4. 视觉件（文本渲染，Markdown 内直接可读）

**Football field 估值图**（估值汇总章必备）：

```text
反向DCF(证伪)      │────市场隐含────│
情景加权DCF            ▓▓▓▓▓▓▓▓▓ 170———225
三要素/EPV          ▓▓▓▓▓▓ 150———195
相对估值               ▓▓▓▓▓▓▓▓ 165———215
EVA/剩余收益          ▓▓▓▓▓▓▓ 160———205
综合区间 [L———————H]        现价 P ▼
```

用等宽代码块 + ▓/─ 字符按比例画；现价用 ▼ 标注。若输出 .docx/.xlsx，可改用真实图表。

**敏感性热力标注**：3×3 表中对高于现价的单元格加粗，让"什么假设下现价合理"一眼可见。

**桥式表**（收入桥/公允价值变动桥）：起点 → 各驱动分项(+/−) → 终点，一行一driver。

## 5. 交付物清单

**原则：交付给用户的只有报告文件本身**；一切中间产物（JSON、脚本输出、CSV）为内部留档，不进入交付清单。

| 产物 | 角色 | 说明 |
|---|---|---|
| 九章报告 **PDF** | **交付物（默认）** | 用户未指明格式时一律交付 PDF；由 .md 源稿转换（pdf 技能或 md→PDF 工具链），确认中文字体与表格渲染 |
| 报告 .md / .docx / .xlsx workbook | 交付物（仅用户指明时） | xlsx workbook 含假设/DCF/情景/输出四页（xlsx 技能） |
| Tearsheet | 交付物（内嵌） | 报告头部内嵌，非独立文件 |
| 估值假设 JSON | 内部留档 | dcf.py config，留在工作目录供复算；关键假设以摘要写入报告附录 |
| 检查器输出 | 内部留档 | 结果摘要写入报告附录 |
| 财务 CSV / 脚本原始输出 | 内部留档 | 用户索要时才提供 |

## 6. 长度与密度纪律

- 深度 = 信息密度 × 覆盖完整性，不是字数。每章自查："删掉哪一段读者不受损失？"能删就删。
- 参考量级：完整模式正文 6,000–12,000 字 + 15–25 张表；财报模式 4,000–8,000 字。超出上限通常说明在复述数据而非分析。
- 附录收纳原始数据长表；正文只放被论证使用的数字。

## 7. 置信度自评（第九章末尾必附）

| 维度 | 评分（高/中/低） | 说明 |
|---|---|---|
| 数据完整度 | | 关键数据获取比例、Tier 1 覆盖率 |
| 估值收敛度 | | 多方法区间重叠程度 |
| 预期差清晰度 | | variant view 是否可证伪 |
| 财报可信度 | A/B/C/D | forensic 模块输出 |
| **综合置信度** | | 低置信度 + 好标签 ≠ 大仓位 |
