# 公用事业行业深度研究附录

适用于受监管电力/燃气/水务、公用事业控股公司、独立发电商及具有长期合同的电力基础设施。先区分受监管 rate base、竞争性发电、开发项目和非公用业务。

## 1. 监管与来源

来源优先级：州/国家公用事业监管委员会裁决、FERC/当地监管申报 > 公司法定与财务披露 > EIA/NERC/系统运营商数据 > 专业数据库。美国可用 FERC eLibrary/Data（Form 1、2、3-Q、714、EQR）、EIA 发电/负荷/燃料/零售电价数据、州 PUC rate case 文档和 NERC 可靠性评估。其他市场使用当地能源监管、系统运营商和许可文件。

公司“批准回报率”不等于实际 earned ROE；监管资产不等于自动可回收。必须阅读 rate case 最终裁决、成本分摊、滞后、燃料调整和绩效处罚条款。

## 2. 核心 KPI

| 模块 | KPI | 研究问题 |
|---|---|---|
| 监管资产 | rate base、增长、allowed/earned ROE、equity layer | 增长是否获得及时回收和合理资本结构 |
| 客户/需求 | customers、sales/load、weather-normalized demand | 负荷来自数据中心、人口还是天气 |
| 费率 | customer bills、rate case、fuel adjustment、regulatory lag | 可负担性是否限制投资回收 |
| 运营 | SAIDI/SAIFI、outages、capacity factor、heat rate | 可靠性与维护是否支撑回报 |
| 资本 | capex、CWIP、AFUDC、project schedule/cost | 建设期利息和延期是否抬高账面利润 |
| 资产负债 | FFO/debt、debt maturity、interest coverage、pension | 融资需求与评级压力 |
| 发电 | MW、generation mix、hedges/PPA、merchant exposure | 合同与现货风险是否匹配 |
| 环境 | emissions、decommissioning、nuclear/coal liabilities | 退役和合规成本是否充分计提 |

## 3. Rate-base 与项目模型

```text
监管收入需求 ≈ 运营成本 + 折旧 + 税 + 获准 rate base × allowed return
每股增长 = rate-base 增长 × earned return − 融资稀释 − 监管/项目损失
```

- 按辖区建 rate base、capex、折旧、allowed ROE、equity layer 和 regulatory lag；集团平均值会掩盖弱辖区。
- 资本计划拆维护、可靠性、输配电、发电、环境与增长项目；项目逐项记录批准、预算、在建、投产与回收机制。
- 融资模型必须联动债务、利率、评级阈值、股权发行和股息；高 capex 增长不等于高每股价值。
- 竞争性发电按容量、利用率、热耗、燃料、PPA/套保和现货价建模；不要套用 rate-base 估值。
- 天气影响做正常化；数据中心/电气化负荷需核验排队、合同、并网、建设和客户信用。

## 4. 估值

- 受监管业务：股利折现/DCF + P/E + P/B/earned ROE；用 rate-base 增长、allowed/earned ROE、资本结构和融资稀释驱动。
- 综合集团使用 SOTP，受监管、竞争性发电、可再生开发和非核心资产分别估值。
- 反向估值回答现价隐含的 rate-base 增长、earned ROE、终端 P/E 和股权融资。
- 对利率、信用利差、allowed ROE、项目超支和监管滞后做敏感性；股息率不能替代内在价值分析。

## 5. 财报模式

建立 EPS 桥（天气、费率、rate base、融资、一次性）、资本计划桥、项目里程碑、监管案件、负荷与客户账单、FFO/debt 和股息覆盖。区分 GAAP EPS、调整后 EPS 和监管会计收益。

必须回答：增长是否已获监管批准；实际 ROE 为何偏离 allowed ROE；资本计划会产生每股增值还是融资稀释；客户可负担性与政治风险是否上升；项目延期、野火/风暴、核电或燃料成本由谁承担。

## 6. 红旗与输出要求

红旗：capex 上修但监管批准滞后；earned ROE 长期低于 allowed；AFUDC/监管资产推动利润但现金流恶化；股息增长依赖发股；客户账单增速过高；大型项目反复延期超支；评级接近下调；野火/环境/核退役负债不足；数据中心负荷只有意向没有合同与并网。

每份报告必须增加：辖区监管地图、rate-base/ROE 桥、资本项目时间表、负荷与费率、融资与评级、股息覆盖、SOTP/利率敏感性，以及至少 3 个反证阈值（earned ROE、项目超支、监管滞后、FFO/debt、稀释或客户账单）。
