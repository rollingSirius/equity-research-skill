# 半导体行业深度研究附录

适用于芯片设计、晶圆代工、IDM、存储、设备、材料、EDA/IP、封装测试及半导体系统公司。先按价值链和产品类别拆分；不同环节不能共用同一组周期指标或估值倍数。

## 目录
1. 分类与研究原则
2. 来源层级
3. 价值驱动与 KPI
4. 模型与估值
5. 财报模式
6. 护城河、红旗与输出要求

## 1. 分类与研究原则

- **Fabless/加速器/CPU**：看产品性能、软件生态、系统价值量、设计中标、代工与先进封装供给。
- **Foundry/IDM**：看制程节点、良率、产能利用率、晶圆 ASP、资本强度和折旧周期。
- **Memory**：看 bit shipment、bit growth、ASP、库存、供给纪律和单位成本，不能只看收入同比。
- **Equipment/Materials/EDA/IP**：看 backlog、订单、客户 capex、服务收入、装机基数、出口管制和客户集中。
- **OSAT/测试**：看先进封装占比、利用率、单机价值量、良率和客户认证。
- AI、汽车、工业、手机、PC 等终端周期不同；按终端、地区、客户和产品拆分，不把行业总增速直接套到公司。

## 2. 来源层级

1. 监管申报、公司财报/演示、客户与供应商原始披露。
2. WSTS/SIA 的销售、单位和 ASP 统计；SEMI/SEAJ 的设备与 fab 数据；各国海关及 USGS 的材料数据。
3. 出口管制与产业政策使用 BIS、欧盟、各地监管机构原文。
4. Gartner、Omdia、TechInsights、TrendForce、Counterpoint、IDC、Yole 等商业研究可补充份额与价格，但必须注明样本、口径和发布日期。
5. 拆机、渠道与供应链消息只作线索；至少用公司、客户或产业数据交叉验证。

公允要求：WSTS 月度数据可能采用移动平均；订单可能重复、取消或跨期；公司“AI revenue”定义不一。任何份额和 TAM 都写清分母。

## 3. 价值驱动与 KPI

```text
价值 = 终端需求 × 内容量/份额 × ASP × 毛利率
     − 研发与资本强度 − 库存/产能错配 − 地缘与客户集中风险
```

| 环节 | 必看 KPI | 关键陷阱 |
|---|---|---|
| Fabless/系统芯片 | units、ASP、产品 mix、design wins、socket/share、软件/服务收入 | 把预告 design win 当已实现收入；渠道库存重复计算 |
| Foundry/IDM | wafer shipments、utilization、node mix、yield、wafer ASP、capex/depreciation | 产能满载不等于高回报；补贴与客户预付款扭曲 FCF |
| Memory | bit shipment、bit growth、ASP、cost/bit、库存天数、供需增量 | 收入增长可能只来自价格；高点利润不能永续化 |
| Equipment | bookings、backlog、book-to-bill（若仍披露）、WFE、service mix | backlog 取消/推迟；客户重复下单；出口限制 |
| EDA/IP | ARR/订阅、RPO、royalty、design activity、renewal | SaaS 指标与版税周期混在一起 |
| OSAT | utilization、先进封装收入、ASP、yield、capex | 普通封装与先进封装估值混用 |

所有公司还需跟踪：客户/代工厂集中、库存与采购承诺、毛利率桥、研发强度、资本开支、折旧、净营运资本、出口受限收入和产品路线图兑现率。

## 4. 模型与估值

- 先按产品/终端建 `units × ASP × share`，或按 wafer starts × utilization × wafer ASP；避免单一收入增速外推。
- 库存周期用“公司库存天数 + 客户/渠道库存 + 行业 shipments/ASP”三层验证。
- 毛利率桥至少拆价格、产品 mix、良率、代工/材料成本、折旧和一次性库存减值。
- Capex、折旧和产能投放做 5–10 年桥；重资产公司 DCF 必须检查跨周期 ROIC。
- 估值组合：反向 DCF + 三情景 DCF + 跨周期 P/E/EV-EBITDA/EV-Sales；Foundry/IDM 可加 P/B、重置价值；周期底部可用正常化盈利，禁止以峰值利润套中枢倍数。
- 多业务公司按设计、制造、网络、软件或服务 SOTP；净现金与战略投资单列，避免重复计入。

情景必须覆盖：终端需求、份额、ASP/价格周期、毛利率、出口限制、供给瓶颈和 capex 回报。概率不能只由管理层指引决定。

## 5. 财报模式

财报后建立四个桥：收入（units/ASP/mix）、毛利率（价格/良率/成本）、库存（公司/客户/渠道）、供给（产能/先进封装/HBM/设备）。

必须回答：订单与终端消耗是否一致；增量来自结构份额还是周期补库；下一代产品量产、良率和客户采用是否按计划；capex/采购承诺是否领先真实需求；出口管制影响是否转移而非消失。

## 6. 护城河、红旗与输出要求

护城河从 IP/软件生态、工艺学习曲线、认证周期、规模采购、装机服务网络、数据/工具链锁定和切换成本评分。技术领先必须能体现为份额、ASP、毛利率或资本回报。

红旗：库存与应收同时上升；backlog 强但客户 capex 下修；毛利率靠短缺定价而非成本/产品；高额 capex 与利用率下降并存；路线图反复延迟；单一客户/代工厂/地区暴露过高；“AI”收入缺乏定义；峰值利润下大额回购。

每份报告必须增加：价值链定位表、产品/终端收入桥、库存与供给周期表、路线图与验证节点、跨周期估值表，以及至少 3 个可推翻主结论的阈值（份额、毛利率、库存、良率、产品延迟或 capex 回报）。
